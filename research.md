# FM2note 调研报告：播客（小宇宙）→ 笔记（Obsidian）自动化管线

## 一、项目目标

将小宇宙 FM 上订阅的播客，在更新后半天内自动转化为全文文字稿，写入 Obsidian 知识库。核心要求：**全自动化、稳定可靠、成本可控**。

---

## 二、整体架构概览

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  数据源层     │ →  │  音频获取层   │ →  │  转写层       │ →  │  输出层       │
│ RSS/API 监听  │    │  下载音频文件  │    │  语音转文字   │    │  写入 Obsidian│
│ 新剧集检测    │    │  元数据提取   │    │  (ASR)        │    │  Markdown     │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

后续可选扩展：在转写层与输出层之间插入 **AI 提炼层**（摘要、要点提取、思维导图）。

## 三、数据源层：如何获取小宇宙播客内容

### 3.1 方案对比

| 方案                     | 原理                                                                       | 优点                                           | 缺点                                                  | 推荐度     |
| ------------------------ | -------------------------------------------------------------------------- | ---------------------------------------------- | ----------------------------------------------------- | ---------- | -------------------------------- |
| **RSSHub 自建路由**      | RSSHub 已内置 `/xiaoyuzhou/podcast/:id` 路由，直接从 xiaoyuzhoufm.com 抓取 | 标准 RSS 格式，生态成熟，可自部署              | 依赖网页结构，可能被反爬；需维护 RSSHub 实例          | ⭐⭐⭐⭐⭐ |
| **小宇宙原生 RSS**       | 小宇宙支持通过 OPML 导出或联系客服获取 RSS 地址（feed.xyzfm.space）        | 官方支持，稳定                                 | 获取流程较繁琐（需邮件申请或 OPML 导出）              | ⭐⭐⭐⭐   | （违背了本项目自动化工作流原则） |
| **xyz-dl (Python CLI)**  | 通过逆向小宇宙 API，使用 refresh_token 认证直接调取数据                    | 功能最全：支持下载音频、字幕、元数据、批量操作 | 需要认证凭证（refresh_token + device_id），有封号风险 | ⭐⭐⭐⭐   | （有封号风险违背了第一性原理）   |
| **ultrazg/xyz (Go API)** | Go 实现的完整小宇宙 API 封装                                               | 接口丰富（搜索、订阅、评论、播放进度等）       | 当前登录功能标记为不可用，维护不确定                  | ⭐⭐⭐     | （已确认不可用就不再考虑）       |
| **Podwise 第三方服务**   | SaaS 服务，内部使用 RSSHub 获取小宇宙内容                                  | 开箱即用，支持导出到 Obsidian                  | 免费版仅 4 集/月，付费版起步 $5.9/月                  | ⭐⭐⭐     |

### 3.2 推荐方案：RSSHub + 原生 RSS 双保险

**首选：自部署 RSSHub**

RSSHub 是开源 RSS 聚合项目（github.com/DIYgod/RSSHub），其中 `/xiaoyuzhou/podcast/:id` 是最热门路由之一（5万+请求）。

- 路由格式：`https://你的RSSHub域名/xiaoyuzhou/podcast/{播客ID}`
- 播客 ID 从小宇宙播客页面 URL 中提取：`https://www.xiaoyuzhoufm.com/podcast/{ID}`
- 输出标准 RSS 2.0 / Atom / JSON Feed，包含：标题、描述、shownotes、音频文件 enclosure URL、发布时间
- 部署方式：Docker 一行命令，或 Vercel/Railway 免费部署

**备选：小宇宙原生 RSS**（我实测最新版本这是走不通的）

- 在小宇宙 App 中：个人主页 → 订阅 → 右上角分享 → 导出 OPML
- 用文本编辑器打开 OPML 文件即可看到每个播客的 RSS 地址
- 也可以联系客服通过邮箱获取特定播客的 RSS 地址

### 3.3 RSS 数据中可获取的字段

一个标准播客 RSS feed 的 item 通常包含：

```xml
<item>
  <title>节目标题</title>
  <description>Show Notes (HTML 格式)</description>
  <pubDate>发布日期</pubDate>
  <enclosure url="https://xxx.mp3" type="audio/mpeg" length="12345678"/>
  <itunes:duration>01:23:45</itunes:duration>
  <itunes:summary>摘要</itunes:summary>
  <link>小宇宙节目页面链接</link>
</item>
```

**关键信息**：`enclosure.url` 就是音频下载地址，这是整个管线的核心数据。

### 3.4 xyz-dl 工具补充说明

GitHub 项目：`shiquda/xyz-dl`（Python 3.13+）

特别值得注意的是它支持 **字幕下载**（`--with-subtitles`）。这意味着部分小宇宙播客可能已有内置字幕/转写文本。如果目标播客有字幕，可以直接下载文字稿，**完全跳过 ASR 转写步骤**，大幅降低成本和复杂度。

认证方式：需要 `refresh_token` 和 `device_id`，通过抓包小宇宙 App 获取。

### 4.1 方案对比

| **通义听悟 API**                     | 优秀（大模型增强） | ~0.8-2 元/h（按功能） | 超长音频      | 云 API   | ⭐⭐⭐⭐⭐ | （我实测通义听悟是很厉害的，阿里云百炼准确率不清楚） |

| **科大讯飞语音转写**                 | 优秀               | 4.9-9.9 元            | 5 小时以内    | 云 API   | ⭐⭐⭐     |
| **belle-whisper-large-v3-zh (本地)** | 优秀（中文微调）   | 免费                  | 无限制        | 本地     | ⭐⭐⭐⭐   |

那么typeless呢

### 4.2 推荐方案分析

#### 方案 A：阿里云百炼 ASR（云端，推荐生产使用）

- **模型**：千问3-ASR-Flash-Filetrans
- **特点**：专为长音频（最长12小时）异步识别设计
- **计费**：按输入音频秒数，每秒 25 Token，价格极低
- **格式**：支持 mp3, wav, m4a, aac, ogg, flac 等
- **优势**：阿里云生态，中文识别质量顶级，有 VAD 自动过滤静音
- **SDK**：Python/Java/Go 等多语言 SDK

#### 方案 B：通义听悟 API（云端，功能最丰富）（优先考虑，后续打算生成摘要和章节速览）

- **特点**：不仅转写文字，还可生成摘要、章节速览等（大模型能力）
- **计费**：转写基础 + 大模型功能按需叠加
- **免费**：新用户 90 天免费试用
- **优势**：如果后续要做 AI 提炼，通义听悟一站式解决
- **SDK**：阿里云 OpenAPI SDK

#### 方案 C：本地 faster-whisper（离线，零成本）

- **模型推荐**：`belle-whisper-large-v3-zh`（中文微调版，比原版提升 24-65%）
- **速度**：faster-whisper 比原版 Whisper 快 4 倍，内存更低
- **量化**：支持 int8 量化，进一步加速
- **硬件要求**：large-v3 模型约需 10GB 显存（GPU），或纯 CPU 运行（较慢）
- **优势**：完全免费，隐私好，无网络依赖
- **缺点**：需要 GPU 硬件；首次下载模型约 3GB

#### 方案 D：OpenAI Whisper API（云端，简单但有限制）

- **价格**：$0.006/分钟（约 ¥2.16/小时）
- **限制**：单次文件最大 25MB，需要分片上传
- **中文质量**：可用，但不如专门的中文模型
- **新模型**：GPT-4o Mini Transcribe $0.003/分钟，更便宜

### 4.3 成本估算（月度）（从目标播客中找几段音频，四条路径都测试一下，我来人工看下哪个更准，优先测试三个云端的，我暂时没有gpu）

假设每月处理 20 集播客，平均每集 1 小时：

| 方案                | 月成本           |
| ------------------- | ---------------- |
| 阿里云百炼 ASR      | ¥25-50           |
| 通义听悟 API        | ¥16-40（仅转写） |
| 本地 faster-whisper | ¥0（需 GPU）     |
| OpenAI Whisper API  | ¥43              |
| 科大讯飞            | ¥98-198          |

### 4.4 优先推荐

1. **有 GPU**：本地 `belle-whisper-large-v3-zh` + `faster-whisper`，零成本
2. **无 GPU / 追求稳定**：阿里云百炼 ASR，便宜且质量好
3. **后续要 AI 提炼**：通义听悟 API，一站式解决转写+摘要

## 五、输出层：写入 Obsidian

### 5.1 方案对比

| 方案                        | 原理                                       | 优点                         | 缺点                          | 推荐度     |
| --------------------------- | ------------------------------------------ | ---------------------------- | ----------------------------- | ---------- |
| **直接写 Markdown 文件**    | Obsidian vault 就是文件夹，直接写 .md 文件 | 最简单，零依赖，跨平台       | 需要知道 vault 路径           | ⭐⭐⭐⭐⭐ |
| **Obsidian MCP (已安装)**   | 通过 MCP 协议操作 Obsidian                 | 已有工具，支持搜索/写入/更新 | 需要 MCP server 运行          | ⭐⭐⭐⭐⭐ |
| **Obsidian Local REST API** | 社区插件，通过 HTTP API 操作 vault         | 功能丰富，支持 CRUD          | 需要安装插件，Obsidian 需运行 | ⭐⭐⭐⭐   |
| **obsidian:// URI**         | 通过 URI scheme 创建笔记                   | macOS/iOS 原生支持           | 功能有限，不适合批量操作      | ⭐⭐       |

### 5.2 推荐方案：直接写文件 + MCP 辅助

**核心方式：直接写 Markdown 文件到 vault 目录**

Obsidian 的 vault 本质上就是一个文件夹，内含 `.md` 文件。直接用 Python/Node.js 创建 `.md` 文件即可，Obsidian 会自动检测文件变化并加载。

这是最可靠的方式，不依赖任何插件、不需要 Obsidian 运行。

**MCP 辅助**：你已安装 Obsidian MCP，可用于：

- 搜索已有笔记避免重复
- 更新/追加内容到已有笔记
- 管理 frontmatter 和标签

（我明白了，按照这样来使用）

### 5.3 笔记模板设计

建议的播客笔记 Markdown 格式：

```markdown

podcast: "{{播客名称}}"
date: { { 发布日期 } }
duration: "{{时长}}"
source: "{{小宇宙链接}}"
tags:
  - podcast
  - { { 播客名称标签 } }
status: unread

# {{节目标题}}

> **播客**：{{播客名称}}
> **日期**：{{发布日期}}
> **时长**：{{时长}}

{{show_notes 内容}}

{{转写文本，按段落分隔}}

```

## 六、自动化调度层

### 6.1 方案对比

| 方案                    | 适用场景         | 优点                   | 缺点                   |
| ----------------------- | ---------------- | ---------------------- | ---------------------- |
| **本地 cron / launchd** | macOS 本地运行   | 简单，无成本，立即可用 | 电脑需开机             |
| **GitHub Actions**      | 云端定时运行     | 免费（公共仓库），可靠 | 需要处理密钥和文件同步 |
| **Docker + cron**       | 自有服务器       | 环境隔离，可复现       | 需要服务器             |
| **Python APScheduler**  | 作为长驻服务运行 | 灵活，可精细调度       | 需要进程管理           |

### 6.2 推荐方案

**本地 macOS 方案（最简单）**：

使用 macOS 的 `launchd` 或简单的 cron 定时任务：

- 每 2-4 小时检查一次 RSS feed
- 发现新剧集 → 下载音频 → 转写 → 写入 Obsidian vault
- 配合 Obsidian 的文件自动同步（iCloud / Obsidian Sync / Git）

**云端方案（更可靠）**：（我有服务器，我要放到服务器里面，本地mac不够可靠，注意这个服务器在跑别的项目了，react+GO+postgre实例）

GitHub Actions 定时工作流：

- 用 cron 表达式设定检查频率（如 `0 */3 * * *` 每 3 小时）
- 在 Action 中运行 Python 脚本
- 使用 Obsidian Git 插件同步到本地

## 十、关键技术细节与注意事项

### 10.1 音频下载

- 播客音频通常是 mp3 格式，单集 50-150MB
- RSS 中 `<enclosure>` 标签直接给出下载 URL
- 部分 CDN 可能有防盗链，需带 User-Agent 或 Referer
- 建议下载后临时存储，转写完成后删除以节省空间

### 10.2 长音频分片

- OpenAI Whisper API 限制 25MB，需要分片（ffmpeg 切割）
- 阿里云百炼支持最长 12 小时，无需分片
- 本地 faster-whisper 无限制，但建议按 30 分钟分片处理以控制内存

### 10.3 去重与增量更新

- 用 RSS item 的 `<guid>` 或 `<link>` 作为唯一标识
- 维护已处理剧集列表（SQLite 或 JSON 文件）
- 每次只处理新增剧集

### 10.4 错误处理

- RSS 获取失败 → 重试 3 次，间隔递增
- 音频下载中断 → 支持断点续传
- ASR 转写失败 → 记录失败任务，下次重试
- 写入 Obsidian 失败 → 保留中间结果（转写文本）

### 10.5 Obsidian 同步

- 如果 vault 在 iCloud 中 → 文件写入后自动同步
- 如果使用 Obsidian Sync → 同样自动同步
- 如果使用 Git → 写入后执行 `git add && git commit && git push`

## 十三、总结与决策建议

| 决策点             | 推荐选择                                   | 理由                           |
| ------------------ | ------------------------------------------ | ------------------------------ |
| 数据源             | RSSHub 自建                                | 标准化、可控、活跃社区         |
| ASR 引擎（有 GPU） | belle-whisper-large-v3-zh + faster-whisper | 零成本、中文质量好             |
| ASR 引擎（无 GPU） | 阿里云百炼 ASR                             | 便宜、中文质量顶级、支持长音频 |
| Obsidian 写入      | 直接写 .md 文件                            | 最简单可靠                     |
| 调度               | macOS launchd                              | 本地运行，简单高效             |
| 编程语言           | Python                                     | 生态丰富，所有依赖库齐全       |

**核心优势**：整套方案可以做到完全自建、零/低月度成本、数据完全自主，同时保持足够的灵活性来逐步增强功能。
